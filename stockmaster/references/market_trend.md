# 模块1：行情趋势判断

## 目标

判断当前A股市场处于四种状态之一：

| 状态 | 趋势 | 温度 | 对应策略 |
|------|------|------|----------|
| 牛+热 | bull | hot | 进攻策略 |
| 牛+冷 | bull | cold | 谨慎乐观 |
| 熊+热 | bear | hot | 保守观望 |
| 熊+冷 | bear | cold | 空仓关注 |

## 判断维度

### 一、牛/熊判断（趋势方向）

综合以下指标，每项满足计1分，总分≥3为牛市，否则为熊市：

| # | 指标 | 牛市条件 | 数据来源 |
|---|------|----------|----------|
| 1 | 上证指数与MA60 | 收盘价 > MA60 且 MA60 向上 | `ak.stock_zh_index_daily(symbol="sh000001")` |
| 2 | 上证指数与MA250 | 收盘价 > MA250 | 同上 |
| 3 | 创业板指与MA60 | 收盘价 > MA60 且 MA60 向上 | `ak.stock_zh_index_daily(symbol="sz399006")` |
| 4 | 近20日涨幅 | 上证指数20日涨幅 > 0% | 计算最近20个交易日收益率 |
| 5 | MACD周线 | 周线MACD柱状线为正或DIF>DEA | 周线数据计算 |

**缠论辅助判断：**

- 日线级别：若当前处于上升趋势（高点和低点依次抬高），加强牛市判断
- 周线级别：若周线形成底分型后向上，确认牛市趋势

#### 顶分型识别方法
顶分型是由3根K线组成，中间一根K线的最高点是三根K线中最高的。识别步骤：
1. 选取连续3根K线
2. 比较三根K线的最高价：若第二根K线最高价 > 第一根和第三根的最高价
3. 同时要求三根K线有重叠区间（并非孤立的尖峰）
4. 顶分型出现后，若后续K线跌破顶分型最低点，确认趋势转折

#### 底分型识别方法
底分型是由3根K线组成，中间一根K线的最低点是三根K线中最低的。识别步骤：
1. 选取连续3根K线
2. 比较三根K线的最低价：若第二根K线最低价 < 第一根和第三根的最低价
3. 同时要求三根K线有重叠区间
4. 底分型出现后，若后续K线突破底分型最高点，确认趋势转折

#### 如何结合分型判断趋势转折
1. **上升趋势中的顶分型**：当价格创新高后出现顶分型，且随后的下跌跌破顶分型最低点，可能预示上升趋势结束
2. **下降趋势中的底分型**：当价格创新低后出现底分型，且随后的上涨突破底分型最高点，可能预示下降趋势结束
3. **分型与MACD配合**：若分型形成时MACD出现背驰（价格创新高但MACD未创新高），则转折信号更强
4. **多级别确认**：日线分型需配合周线分型综合判断，提高准确率

### 二、热/冷判断（市场活跃度）

综合以下指标，每项满足计1分，总分≥3为热，否则为冷：

| # | 指标 | 热市条件 | 数据来源 |
|---|------|----------|----------|
| 1 | 成交量比 | 当日成交量 > 20日均量 × 1.2 | `ak.stock_zh_index_daily` 成交量字段 |
| 2 | 涨跌比 | 上涨家数 / 下跌家数 > 1.5 | `ak.stock_zh_a_spot_em()` 统计 |
| 3 | 涨停数量 | 涨停家数 > 80 | `ak.stock_zt_pool_em(date)` |
| 4 | 换手率 | 全市场平均换手率 > 2% | `ak.stock_zh_a_spot_em()` 换手率字段 |
| 5 | 北向资金 | 当日北向资金净流入 > 0 | `ak.stock_hsgt_north_net_flow_in_em()` |

**缠论辅助判断：**
- 量能与价格配合：量价齐升为热的强确认
- 背驰信号：若量能放大但指数未创新高，属于热但可能接近转折

## 综合评分逻辑

```
牛熊得分 = 牛市指标满足数量（0-5分）
冷热得分 = 热市指标满足数量（0-5分）

如果 牛熊得分 >= 3：趋势 = "bull"
否则：趋势 = "bear"

如果 冷热得分 >= 3：温度 = "hot"
否则：温度 = "cold"
```

## 边界情况处理

- **得分恰好为3**：属于临界状态，报告中标注"边际"，建议使用更保守的相邻策略
- **数据缺失**：某项指标数据获取失败时，该项不计分，总分基数相应减少
- **盘中判断**：交易时间内数据为实时但不完整，报告中注明"盘中预判，收盘后需确认"

### 当得分恰好为3时的临界判断
当牛熊得分或冷热得分恰好为3时，处于临界状态，处理方式如下：
1. **边际标记**：在输出结果中增加 `margin: true` 字段
2. **策略调整**：使用更保守的相邻策略（进攻策略→谨慎乐观，保守观望→空仓关注）
3. **二次确认**：若处于临界状态，建议等待下一个交易日确认信号
4. **分型辅助**：此时缠论分型可作为辅助判断依据，若出现明确的顶/底分型转折信号，可增强判断信心

### 数据获取失败时的容错处理
当某项指标数据获取失败时，系统应保持稳定运行：
1. **单项跳过**：获取失败的指标不计分，但保持其他指标正常评估
2. **基数调整**：总分基数相应减少（如5项指标中2项失败，基数为3，需满足≥2分而非≥3分）
3. **重试机制**：首次获取失败后，间隔5分钟自动重试，最多重试3次
4. **日志记录**：所有数据获取失败需记录日志，包含时间、指标名称、错误原因
5. **降级策略**：若关键指标（如上证指数数据）连续失败，返回"未知"状态并告警

### 盘中与收盘后判断的差异
盘中判断与收盘后判断存在本质区别，需分别处理：

| 差异项 | 盘中预判 | 收盘后确认 |
|--------|----------|------------|
| 数据完整性 | 仅当日部分数据 | 完整日线数据 |
| MA60/MA250 | 实时均价可能波动 | 固定收盘价计算 |
| 成交量 | 当日成交进行中 | 最终成交量确定 |
| 涨停家数 | 盘中逐步增加 | 最终涨停数确定 |
| 北向资金 | 仅盘中有数据 | 收盘后最终数据 |

**盘中判断注意事项：**
- 报告中需标注"盘中预判，最终结果待收盘确认"
- 趋势判断可参考，但分型识别需等待收盘后确认
- 若盘中出现大幅波动（>2%），建议等待市场稳定后再做判断

## 数据来源

本模块使用的 akshare API 汇总：

| API | 用途 | 数据内容 |
|-----|------|----------|
| `ak.stock_zh_index_daily` | 获取上证指数、创业板指等指数日线数据 | 开盘价、收盘价、最高价、最低价、成交量 |
| `ak.stock_market_activity_legu` | 获取市场活跃度数据 | 涨跌家数、成交金额等 |
| `ak.stock_market_fund_flow` | 获取市场资金流向 | 个股资金流向、板块资金流向 |
| `ak.stock_hsgt_fund_flow_summary_em` | 获取北向资金（沪深港通）流向汇总 | 北向资金净流入、沪股通、深股通数据 |

**数据更新频率：**
- 日线数据：收盘后更新（约16:00后）
- 北向资金：盘中实时更新，收盘后确认
- 市场活跃度：盘中实时更新
