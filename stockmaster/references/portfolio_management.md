# 模块6：持仓管理与监控

## 功能概述

持仓管理模块负责：
1. 维护持仓数据（记录买入、卖出、调仓操作）
2. 每日获取持仓股最新行情
3. 结合当前市场策略，对每只持仓股生成操作指令
4. 跟踪账户整体盈亏和风险敞口

## 持仓数据格式

持仓记录保存在 `data/portfolio.json`：

```json
{
  "account": {
    "total_capital": 100000,
    "cash": 60000,
    "last_updated": "2024-01-15"
  },
  "holdings": [
    {
      "code": "600519",
      "name": "贵州茅台",
      "buy_date": "2024-01-10",
      "buy_price": 1680.00,
      "quantity": 100,
      "cost": 168000.00,
      "stop_loss": 1596.00,
      "target_profit": 1932.00,
      "strategy": "cautious",
      "notes": "白马蓝筹，谨慎乐观策略建仓"
    }
  ],
  "watchlist": [
    {
      "code": "300750",
      "name": "宁德时代",
      "reason": "行业龙头，跌幅充分，等待买入信号",
      "added_date": "2024-01-12"
    }
  ],
  "history": []
}
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `code` | 股票代码（6位，含交易所前缀：sh/sz） |
| `buy_date` | 建仓日期 |
| `buy_price` | 平均成本价 |
| `quantity` | 持有数量（股） |
| `cost` | 总成本（buy_price × quantity） |
| `stop_loss` | 止损价（买入时即设定） |
| `target_profit` | 目标止盈价 |
| `strategy` | 建仓时的策略（aggressive/cautious/conservative/defensive） |
| `notes` | 建仓理由备注 |

## 每日监控流程

### Step 1：获取持仓股最新数据

对每只持仓股获取以下数据：

```python
import akshare as ak

# 实时行情
spot_df = ak.stock_zh_a_spot_em()
# 筛选持仓股的行情数据：最新价、涨跌幅、成交量、换手率

# 日K线（计算技术指标）
kline_df = ak.stock_zh_a_hist(symbol="600519", period="daily", adjust="qfq")
```

### Step 2：计算每只持仓股的关键指标

| 指标 | 计算方式 |
|------|----------|
| 当前盈亏% | (最新价 - 成本价) / 成本价 × 100% |
| 距止损位% | (最新价 - 止损价) / 最新价 × 100% |
| 距目标位% | (目标价 - 最新价) / 最新价 × 100% |
| 持有天数 | 当前日期 - 买入日期 |
| MA5位置 | 最新价 vs MA5 |
| MA10位置 | 最新价 vs MA10 |
| MA20位置 | 最新价 vs MA20 |
| MA60位置 | 最新价 vs MA60 |
| 量能状态 | 当日成交量 / 20日均量 |
| RSI(14) | 14日相对强弱指标 |
| MACD状态 | DIF vs DEA，柱状线方向 |

### Step 3：生成操作指令

**操作指令类型：**

| 指令 | 含义 | 颜色标记 |
|------|------|----------|
| 🔴 卖出 | 触发止损或强制卖出信号 | 红色警报 |
| 🟡 减仓 | 部分止盈或风险预警 | 黄色警告 |
| 🟢 持仓 | 趋势正常，继续持有 | 绿色安全 |
| 🔵 加仓 | 回调到位，可以加仓 | 蓝色机会 |
| ⚪ 买入 | 新标的触发买入信号 | 白色提示 |

### Step 4：策略关联的指令生成规则

操作指令的判断必须结合当前市场策略。不同策略下，同样的技术信号可能产生不同的操作指令。

**通用卖出规则（所有策略都适用）：**
- 亏损触及止损价 → 🔴 卖出
- 出现重大利空消息 → 🔴 卖出

**策略差异化规则：**

读取对应策略文档中的"持仓操作指令规则"部分，获取该策略下的具体判断标准。

#### 缠论辅助判断（适用于所有策略）

在技术指标分析基础上，叠加缠论判断：

| 缠论信号 | 操作影响 |
|----------|----------|
| 出现顶分型 | 提高减仓/卖出的优先级 |
| 出现底分型 | 提高加仓/买入的优先级 |
| 中枢震荡中 | 维持持仓，关注中枢突破方向 |
| 向上离开中枢 | 持有，等待背驰信号 |
| 向下离开中枢 | 减仓或卖出 |
| 出现背驰 | 当前方向可能反转，相应调整 |

### Step 5：风险检查

在生成操作指令后，进行整体风险检查：

| 检查项 | 阈值 | 处理 |
|--------|------|------|
| 总仓位是否超限 | 超过当前策略仓位上限 | 提示减仓至合规水平 |
| 单票仓位是否超限 | 超过当前策略单票上限 | 提示减仓该股票 |
| 同行业集中度 | 同行业持仓 > 总仓位40% | 提示分散风险 |
| 账户总亏损 | 总浮亏 > 10% | 发出风险警报 |
| 策略不匹配 | 持仓策略 ≠ 当前策略 | 提示需评估是否调整 |

## 操作记录

每次操作（买入/卖出/加仓/减仓）后，自动记录到 `history` 数组：

```json
{
  "date": "2024-01-15",
  "code": "600519",
  "name": "贵州茅台",
  "action": "sell",
  "price": 1750.00,
  "quantity": 50,
  "reason": "盈利超15%，减仓50%",
  "strategy": "cautious",
  "profit_pct": 4.17
}
```
